ASOS web case study

2017 | 2018 November - May

Main role: UX Designer

Download

Download the complete project

Summary

objectives

  • Increase sales conversion
  • Increase average ticket
  • Find and fix design problems

Solutions

  • Crosselling
  • Stock alerts
  • Redesign of some items

Context

This project took place at KSchool from February to March between the main project of the master (the develop of an app), we were a team of five persons. I did heuristics test, user testing, flow diagrams and help wiht a navigable Axure prototype. While we were working at the beginning in this project we were tracking ASOS web. ASOS updated the web so many times that our overall proposals were being discarded due to they were already implemented, anyway it was fun to compete against them.

Research

Heuristic test

We did a first approach through a heuristic test carried by each member of the team then we put together the results that in the most cases coincided.

This were our find outs:

  • ASOS lacks of strong visual identity at corporative level.
  • Private area should be more customisable.
  • Layout could be more efficient.
  • Web makes you feel insecure at some process.
  • Overall results of ASOS web were good.

User testing

To find errors or sections which should be fixed we made some user testing with the tool Lookback. First, we designed scenarios which various task that should be completed and then we carried the test. There were five users who made this test, it took them about 25 to 40 minutes to complete all the tasks. Some of the insights were:

  • User feels unsecure when selecting payment method
  • The only way to know if a product is out of stock is by clicking on the product card.
  • Selecting a delivery type is not easy
  • Navigation not too much intuitive

Desing process

Flow diagram and guide wireframes

To help us in the design process we developed a flow diagram. I was in charge of this piece, as it was the first flow diagram I had ever had developed I made a big one with all tasks but later I learnt that is better to make small flow diagrams of each task to evade complexity. We also delivered a guide wireframe which we used for explaining the overall approach to our client. We also did a value proposal canvas.

Problems and solution

*A few examples, for more info please download the project

Product card

  • When you have found a product without stock the user flow will stop, there were not other possible actions resulting in a lost and frustrated user.
    Solution: bring a new way to save a notification that would alert the user when stock were avalaible again.
  • Icon used for share a product was confused by an upload icon.
    Solution:Change the icon to another more recognisable.

Solution

Whislist icon

  • Wishlist icon were still the same if you had products saved or not.
    Solution: change the icon when you save products. We also add an indicator number at the icon to see how much items you saved

Solution

First iteration

To test all our design solutions we developed an Axure mockup and tested it with our users and we found out the next problems:

  • Buttons "Add to bag" and "Wishlist" should have an animation every time an action occur to reinforce it (example: a shake).
  • At the product card, change radio buttons of stock alerts for normal text buttons.
  • Improve the design of pay methods section at the checkout.
